Controlling Program Execution

As soon as the debugger attaches to a program (or a new program is executed from the debugger), the execution of the debuggee is stopped.

At this point, you can:

  • insert breakpoints
  • single step through the program
  • continue executing the program

    • Note: Most operations require the debugged program to be stopped in the debugger. For example, in order to insert or remove a breakpoint, you need first to select Program -> Break (or use the Break button from the toolbar). Once the program stops, you may proceed to the Breakpoints menu.


    Single Stepping

    You can execute the program, one step at a time, at either machine level or source level. At machine level, you can execute one CPU instruction at a time. This functionality can be accessed by selecting Step Instruction from the Program menu.

    To execute one line of source code, select Step from the Program menu, or press the Step button on the toolbar (if the toolbar is not visible, turn it on by selecting Toolbar in the View menu).

    While stepping at the source level, if a function call is encountered, the debugger will step inside of that function. If you want to step over function calls (i.e. not step inside called functions) use the Next menu from the Program menu on the menu bar, or use the Next button from the toolbar.

    Note: When running in text mode (no graphical user interface), the following commands are available at the command prompt:

  • instruction
  • next
  • step

    • These work esentially the same as the menus; for more information, type:
    help <command> at the prompt.


    Resuming Execution

    To continue executing the program, select Continue from the Program menu. The execution will resume until either a breakpoint or watchpoint is hit, the program receives a signal, or you stop it. To stop the program, select Break from the Program menu, or from the toolbar.

    The Return from function menu (under Program) and the Return toolbar button allow you to resume the execution of the debugged program until the current function returns to its caller. As soon as the function returns, the program will break into the debugger.

    To continue executing the program until a given line of code is reached, right-click on that line and select the Run To Cursor menu option.
