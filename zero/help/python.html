Controlling the Debugger with Python

The Zero debugger ships with a plugin (zpython.so) embedding a Python interpreter.

The interpreter is augmented with the zero module, which provides access to the fundamental building blocks of the debugger. One can access artifacts such as zero.Debugger, zero.Process, zero.Thread, zero.BreakPoint, and so on.

The plugin also arranges for user-defined callback functions to be invoked when interesting events (such as a breakpoint being hit, for example) occur.

There are several ways to feed Python scriptlets to the debugger:

  • specify a Python source file at the command line, like so: zero --py-run=sourcefile;
  • put the code in a hidden file named .zero.py, in the current directory;
  • put the code in a hidden file named .zero.py in the plugins directory (as specified by the ZERO_PLUGIN_PATH environmental variable).;
  • type the code interactively in the Python Console
    Note: the Console is not available in the BETA and DEMO versions of the program).


    • Note: The .zero.py file that currently ships with the debugger (and is installed in the plugins directory) implements the on_debug_symbol callback. This allows STL string, set and vector objects to be displayed in a more intuitive way. Users may extend this file with their own custom visualisations for various data types. The callback acts as a filter, returning None if no transformation should be applied.
