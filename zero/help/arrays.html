Evaluating Array Ranges

In order to avoid linear complexity in displaying large arrays, the ZeroBUGS' built-in expression interpreter extends the C/C++ language rules with a new postfix grammar rule.

Given an array myArray, you can inspect a range of values by evaluating any of the following expressions:

  1. myArray[low:high]
  2. myArray[:high]
  3. myArray[low:]
where low and high can be any legal expressions that evaluate to a value of integer type.

In addition, you can inspect any individual element of the array, by evaluating myArray[index], where index is some expression that evaluates to an integer.

The interpreter limits internally the number of elements that are shown when none of the above subscript postfix expressions are used. The value is determined by the ZERO_MAX_ARRAY environment variable.
