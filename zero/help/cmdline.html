Command Line Options

Synopsis:
zero [progname | corefile] [arg [arg]]

arg may be either a command line option for the debugger, or an argument for the debugged program. Arguments that follow the program name, and do not start with a dash, are passed as command line arguments to the debugged program.

If you need to pass to the debuggee an argument that contains spaces, then you need to put double quotes around it.

Arguments starting with a dash are passed to the debugger engine and to the plug-ins. If you need to pass command line arguments starting with a dash to the debugged program, prefix them with a standalone, double dash. For example: zero foo -v --main passes -v and --main to the debugger engine, while: zero foo -- -v --main passes -v and --main to the debugged program foo.

Debugger engine command line options:

  • --break-on-throw: stop target execution before throwing an exception
  • --cache-disable: do not cache debug information from one run to another.
  • --main: automatically position at main, and start debugging from there, skipping the startup code. This is not the default, since it does not apply to: programs that don't have a main function, programs that have not been executed from under the debugger, and post-mortem debugging. This is equivalent to setting the ZERO_START_MAIN environment variable to true.
  • --no-banner: do not display copyright and version info to console when starting up the debugger
  • --no-hw-bp(no hardware breakpoints): disable usage of hardware breakpoints, works around a bug in Virtual PC. This is equivalent to setting the ZERO_HARDWARE_BREAKPOINTS environment variable to zero (or false).
  • --no-trace-fork: do not trace into forked processes
  • --syscall-break: break into the debugger each and every time the debugge is about to enter and leave a system call.
  • -v: development builds only: print verbose information to console.
  • --dwarf-nofh: disable frame handlers in the dwarf plugin; frame handlers read stack unwinding info from the debuggee and work with the stack unwinder. Frame handlers are enabled by default.
  • --ui-disable: do not activate the UI plugin even if found in the plugin path
